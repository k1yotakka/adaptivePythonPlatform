# Документация серверной части проекта PyLearn

## Используемые технологии

Серверная часть реализована на Python 3.11+ с использованием следующих библиотек и инструментов:
- FastAPI — современный веб-фреймворк для построения REST API
- SQLAlchemy — ORM для работы с базой данных
- PostgreSQL — реляционная база данных
- Pydantic — валидация входных и выходных данных
- JWT (PyJWT) — токены для аутентификации
- bcrypt (passlib) — безопасное хеширование паролей
- httpx — асинхронный HTTP-клиент для работы с Gemini API

Для запуска сервера используется uvicorn с автоматической перезагрузкой:
```
python run.py
```

После запуска сервер доступен по адресу http://localhost:8000, а интерактивная документация Swagger UI — по адресу http://localhost:8000/docs.

## Схема базы данных

В проекте используется PostgreSQL со следующими основными таблицами:

Таблица users:
- Хранит данных пользователей всех ролей (студенты, преподаватели, администраторы)
- Основные поля: id, name, email, password_hash, role, level, learning_goal, avatar_url
- Роль определяет уровень доступа (student / teacher / admin)
- Уровень (level) указывается только для студентов (beginner / intermediate / advanced)

Таблица courses:
- Курсы, создаваемые преподавателями
- Связь: teacher_id → users.id

Таблица modules:
- Модули внутри курсов, упорядочены по полю order
- Связь: course_id → courses.id

Таблица lessons:
- Уроки внутри модулей, содержат теоретический материал и задачи
- Связь: module_id → modules.id

Таблица tasks:
- Задачи двух типов:
  - lesson — привязаны к урокам курса
  - standalone — отдельные задачи в стиле HackerRank
- Поля: title, description, difficulty, task_type, level, topic, starter_code, expected_output

Таблица groups:
- Группы студентов, каждая имеет уникальный invite_code
- Связь: teacher_id → users.id, course_id → courses.id

Таблица enrollments:
- Связь студентов с группами (многие ко многим)
- Поля: student_id, group_id

Таблица submissions:
- Отправленные решения задач
- Поля: student_id, task_id, code, score, is_correct, ai_feedback

Таблица achievements:
- Достижения студентов
- Поля: user_id, key (уникальный идентификатор), name, emoji

Связи между сущностями реализованы через внешние ключи с каскадным удалением там, где это необходимо.

## Аутентификация и авторизация

Система использует JWT-токены для аутентификации пользователей. После успешной регистрации или входа сервер возвращает токен, который клиент должен передавать в заголовке Authorization: Bearer <token> при каждом запросе.

Токены создаются функцией create_access_token из модуля auth.py и содержат идентификатор пользователя в поле sub. Срок жизни токена по умолчанию 24 часа (настраивается через ACCESS_TOKEN_EXPIRE_MINUTES в .env).

Пароли хешируются с использованием bcrypt перед сохранением в базу данных.

Зависимости FastAPI для проверки прав доступа:
- get_current_user — получает текущего пользователя из токена, доступно всем авторизованным
- require_teacher — разрешает доступ только преподавателям и администраторам
- require_admin — разрешает доступ только администраторам

## API эндпоинты

Все эндпоинты доступны по базовому адресу http://localhost:8000/api/v1

### Аутентификация и профиль (/auth)

POST /auth/register
- Регистрация нового пользователя
- Тело: name, email, password, role, learning_goal (опционально)
- Возвращает: access_token и данные пользователя

POST /auth/login
- Вход в систему
- Тело: email, password
- Возвращает: access_token и данные пользователя

GET /auth/me
- Получение данных текущего пользователя
- Требует авторизации

PATCH /auth/me/level
- Установка уровня для студента (beginner/intermediate/advanced)
- Тело: level

PATCH /auth/me/goal
- Установка цели обучения
- Тело: learning_goal

PATCH /auth/me/profile
- Обновление профиля пользователя (ФИО, email, пароль)
- Тело: name (опционально), email (опционально), password (опционально)
- Проверяет уникальность email и минимальную длину пароля (6 символов)

POST /auth/me/avatar
- Загрузка аватара пользователя
- Тело: multipart/form-data с файлом изображения
- Сохраняет файл в static/avatars/ и возвращает обновленного пользователя с avatar_url

### Курсы (/courses)

GET /courses/
- Список курсов (преподаватель видит свои, студент — курсы своих групп)

POST /courses/
- Создание нового курса (только преподаватели)
- Тело: title, description

GET /courses/{id}
- Получение курса по ID

PATCH /courses/{id}
- Обновление курса (только автор курса)

DELETE /courses/{id}
- Удаление курса (только автор)

GET /courses/{course_id}/modules
- Список модулей курса

POST /courses/{course_id}/modules
- Создание модуля в курсе
- Тело: title, description, order

PATCH /courses/modules/{id}
- Обновление модуля

DELETE /courses/modules/{id}
- Удаление модуля

GET /courses/modules/{id}/lessons
- Список уроков в модуле

GET /courses/modules/{id}/lesson-progress
- Прогресс студента по урокам модуля (какие задачи решены)

POST /courses/modules/{id}/lessons
- Создание урока в модуле
- Тело: title, content, order

GET /courses/lessons/{id}
- Получение урока со всеми задачами

PATCH /courses/lessons/{id}
- Обновление урока

DELETE /courses/lessons/{id}
- Удаление урока

### Задачи (/tasks)

GET /tasks/
- Список задач с фильтрацией по типу (?task_type=standalone или lesson)
- Студенты видят только задачи своего уровня и standalone-задачи
- Преподаватели видят только свои задачи

GET /tasks/topics
- Список всех уникальных топиков задач (loops, functions, oop и т.д.)

POST /tasks/
- Создание новой задачи (только преподаватели)
- Тело: title, description, difficulty, task_type, level, topic, starter_code, expected_output

GET /tasks/{id}
- Получение задачи по ID

PATCH /tasks/{id}
- Обновление задачи (только автор)

DELETE /tasks/{id}
- Удаление задачи (только автор)

POST /tasks/{id}/submit
- Отправка решения задачи
- Тело: code
- Выполняет код студента, сравнивает вывод с expected_output
- Возвращает результат проверки (is_correct, score)
- При правильном решении начисляются достижения

GET /tasks/{id}/status
- Статус задачи для текущего студента (не начата / в процессе / решена)

GET /tasks/{id}/my-submissions
- История отправок студента по данной задаче

### Студенты (/students)

GET /students/dashboard
- Данные для главной страницы студента
- Возвращает: streak, количество решенных задач, курсы, последние отправки

GET /students/progress
- Статистика прогресса студента по темам

GET /students/achievements
- Список полученных достижений студента

GET /students/available-achievements
- Все возможные достижения с текущим прогрессом

### Практика (/practice)

POST /practice/run
- Выполнение Python-кода в безопасном окружении
- Тело: code, timeout (по умолчанию 5 секунд)
- Возвращает: stdout, stderr, exit_code, success

Код запускается в отдельном процессе через subprocess с ограничением по времени.

### AI-помощник (/ai)

POST /ai/feedback
- Запрос подсказки от AI (Gemini API)
- Тело: task_description, student_code, question (опционально)
- Если указан вопрос — AI отвечает на конкретный вопрос
- Если вопроса нет — AI даёт структурированный фидбек по 4 блокам

### Группы (/groups)

GET /groups/
- Список групп преподавателя

POST /groups/
- Создание новой группы
- Тело: name, course_id (опционально)
- Генерирует уникальный invite_code

GET /groups/{id}
- Получение группы по ID

DELETE /groups/{id}
- Удаление группы

GET /groups/{id}/students
- Список студентов группы

GET /groups/{id}/stats
- Статистика студентов группы (решенные задачи, средний балл)

POST /groups/join
- Запись студента в группу по invite-коду
- Тело: invite_code

### Администрирование (/admin)

GET /admin/teachers
- Список всех преподавателей (только для администратора)

POST /admin/teachers
- Создание нового преподавателя (только администратор)

DELETE /admin/teachers/{id}
- Удаление преподавателя (только администратор)

GET /admin/students
- Список всех студентов (только администратор)

## Интеграция с Gemini API

Для генерации подсказок используется Gemini API от Google. При первом запросе система запрашивает список доступных моделей и выбирает наиболее приоритетную из списка PREFERRED_MODELS (gemini-2.5-flash, gemini-2.0-flash, gemini-1.5-flash).

Выбранная модель кэшируется в памяти. Если при запросе модель возвращает ошибку 404 (модель недоступна), система автоматически обновляет список и выбирает другую.

Предусмотрена система повторных попыток (retry) при временных ошибках (503, 429) с экспоненциальной задержкой.

Промпт для AI построен так, чтобы не выдавать студенту готовое решение, а направлять его мысли в правильное русло. Ответ структурируется по четырём блокам:
1. Понимание задачи
2. Что сделано правильно
3. Что нужно исправить (+ легкая логическая подсказка)
4. Следующий шаг

## Проверка решений задач

Функция _evaluate_submission в модуле routers/tasks.py выполняет следующие действия:

1. Создает временный .py файл с кодом студента
2. Запускает файл через subprocess с таймаутом 10 секунд
3. Получает stdout и сравнивает с expected_output из задачи

Сравнение вывода учитывает три случая:
- Точное совпадение строк
- Повторение ожидаемого вывода несколько раз (если студент случайно запустил код несколько раз или добавил демонстрационные print)
- Совпадение конца вывода с ожидаемым результатом (если в начале есть отладочная печать)

Это позволяет избежать ложных отрицательных результатов из-за технических особенностей выполнения кода.

## Система достижений

Функция _award_achievements автоматически начисляет достижения студентам после успешной отправки задачи:

Достижение "First Steps" (👣):
- Выдается за первое правильное решение любой задачи

Достижения "{Topic} Master":
- Выдаются за решение всех standalone-задач по определенной теме
- Учитывается уровень студента (считаются только задачи соответствующего уровня)
- Эмодзи для каждой темы настраиваются в словаре TOPIC_EMOJI

## Тест на определение уровня (screening test)

Вопросы для screening test хранятся на фронтенде в файле frontend/src/pages/Student/Onboarding/ScreeningTest.jsx в виде массивов INTERMEDIATE_TASKS и ADVANCED_TASKS.

Каждая задача содержит:
- starterCode — заготовка кода
- testCode — код проверки, который выполняется после кода студента
- expectedOutput — ожидаемый результат testCode

При выполнении testCode окружается специальными маркерами (__PYLEARN_HARNESS_START__ и __PYLEARN_HARNESS_END__), чтобы отделить вывод теста от вывода кода студента.

## Настройка окружения

Все чувствительные данные и параметры конфигурации хранятся в файле .env:

```
DATABASE_URL=postgresql+psycopg2://postgres:password@localhost:5432/pylearn
SECRET_KEY=your-secret-key-here
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=1440
CORS_ORIGINS=http://localhost:5173,http://localhost:5174,http://localhost:5175
ADMIN_EMAIL=admin@pylearn.com
ADMIN_PASSWORD=admin123
GEMINI_API_KEY=your-gemini-api-key
```

SECRET_KEY используется для подписи JWT-токенов и должен быть уникальным для каждой установки.

CORS_ORIGINS содержит список разрешенных доменов фронтенда (для разработки обычно localhost с разными портами).

GEMINI_API_KEY получается в Google AI Studio (https://aistudio.google.com/app/apikey).

## Миграции базы данных

Проект использует упрощенный подход к миграциям. При запуске приложения автоматически выполняются:

1. Base.metadata.create_all(bind=engine) — создание всех таблиц по моделям
2. run_migrations() — добавление новых столбцов к существующим таблицам

Функция run_migrations использует SQL-команды с конструкцией IF NOT EXISTS, что позволяет безопасно добавлять новые поля без риска ошибок при повторном запуске.

Пример:
```sql
ALTER TABLE users ADD COLUMN IF NOT EXISTS avatar_url VARCHAR(300)
```

Это простое решение подходит для дипломного проекта, хотя в промышленной разработке рекомендуется использовать Alembic или аналогичные инструменты миграций.

## Типичный сценарий работы студента

1. Регистрация через POST /auth/register
2. Прохождение онбординга (выбор цели и уровня)
3. Вступление в группу по invite-коду через POST /groups/join
4. Просмотр доступных курсов GET /courses/
5. Изучение модулей и уроков курса
6. Решение задач с использованием POST /practice/run (тестовый запуск) и POST /tasks/{id}/submit (отправка решения)
7. Получение AI-подсказок через POST /ai/feedback
8. Отслеживание прогресса через GET /students/dashboard и /students/progress
9. Редактирование профиля и смена аватара через /auth/me/profile и /auth/me/avatar

Все данные о прогрессе, решениях и достижениях сохраняются в базе данных и доступны через соответствующие эндпоинты.
